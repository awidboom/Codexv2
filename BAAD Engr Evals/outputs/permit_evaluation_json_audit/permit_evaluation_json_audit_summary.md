## Backfill Notes

- The `*_raw_candidate` columns only turn `yes` when the master section is empty and relevant raw text exists.
- `*_raw_alias_candidate` is the high-confidence signal: the parser already isolated a raw block into a matching section alias.
- `*_raw_unmapped_candidate` is a weaker signal from keyword matches in `raw_sections.unmapped_blocks`.
- For `conditions` and `permit_conditions`, the same raw alias can support both the narrative section and structured item extraction.
- Review the per-file CSV first, then sort by `missing_sections_with_raw_candidates` to prioritize manual or scripted backfill work.
